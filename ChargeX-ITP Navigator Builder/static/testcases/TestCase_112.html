Test NameValid PKI certificates, expired CRL/OCSP cache outside 4 weeks for SubCA Certificates [Multiple enumerations] [NREL 1.2.3]
Test IdentifierITP-03-16-003
Test TypeRevocation: Sub-CA2 Certs [NREL Test Cases]
Test CategorySingle PKI Tests
Test PurposeTo ensure the EVSE does not initiate a charging session without confirming the validity of the PKI certificates.
Observed MetricsSession Initialization, Messages, EV UI response, EVSE UI response, HLC Message Sequence
Intended MRECs/Errors--
Other Possible MRECsInvalidToken, InvalidPayment
Test ConditionsAuthentication Type/sPnC
HLC Protocol/sISO 15118-2, ISO 15118-20
Involved System/sEV, EVSE
Certificate/s (PKI)Valid Certificate/sV2G Root CA Cert, EV Provisioning Cert, SECC Leaf Certificate, Contract Cert, EV SubCA1 Cert, EV SubCA2 Cert, CSO SubCA1 Cert, CSO SubCA2 Cert, (Optionally: ProvServ Cert)
Invalid Certificate/sNone
Reason/s for Certificate/s Invalidity1. Expired cached CRL (older than 4 weeks), no access to OCSP
2. Expired OCSP response for SubCA2 Cert, not possible to send new OCSP request
Steps1. Ensure 'Valid Certificate/s" are installed on the appropriate entity within the charging environment.
2. Ensure all certificates form a valid chain to a common trust anchor.
3. Ensure the cached CRL/ cached OCSP response is dated before 4 weeks. (Manually alter the date of the cache stored)
4. Take the EVSE offline to disable OCSP connections and prevent it from downloading an updated CRL.
5. Set EVSE HLC highest priority to 'HLC Protocol'.
6. Set EVSE authentication option to 'Authentication Type'.
7. Plug-in vehicle.
8. If power transfer begins, terminate the charge session 30-60s into power transfer.
9. Unplug EV.
Pass Criteria (Check box if met)
• EVSE tries to get an updated CRL by accessing the CRL Distribution point provided in the certificate metadata.
• With no or hanging OCSP response, EVSE cannot get an updated certificate validity information.
• The charge session is not initiated.
• EVSE prompts the user to use EIM without unplugging the EV.

Test NameCRL to verify PKI certificates and check revocation [NREL 1.1.1]
Test IdentifierITP-03-15-001
Test TypeRevocation: EVSE & Contract Certs [NREL Test Cases]
Test CategorySingle PKI Tests
Test PurposeTo ensure EVSE can retrieve, validate and use a Certificate Revocation List (CRL) from a CA or SubCA.
Observed MetricsSession Initialization, Messages, EV UI response, EVSE UI response, HLC Message Sequence
Intended MRECs/Errors--
Other Possible MRECs--
Test ConditionsAuthentication Type/sPnC
HLC Protocol/sISO 15118-2, ISO 15118-20
Involved System/sEV, EVSE, PKI Provider
Certificate/s (PKI)Valid Certificate/sV2G Root CA Cert, EV Provisioning Cert, SECC Leaf Certificate, Contract Cert,(Optionally: EV SubCA1 Cert, EV SubCA2 Cert, CSO SubCA1 Cert, CSO SubCA2 Cert, ProvServ Cert)
Invalid Certificate/sNone
Steps1. Ensure 'Valid Certificate/s' are installed on the appropriate entity within the charging environment.
2. Ensure all certificates form a valid chain to a common trust anchor.
3. Disable OCSP connections to ensure the EVSE depends solely on CRL retrieval for certificate validation.
4. Ensure the Certificate Revocation List (CRL) is updated and none of the valid PKI certificates are listed in the CRL.
5. Set EVSE HLC highest priority to 'HLC Protocol'.
6. Set EVSE authentication option to 'Authentication Type'.
7. Plug-in vehicle.
8. Observe the session initialization into power transfer.
9. Terminate charge session 30-60s into power transfer.
10. Unplug EV.
Pass Criteria (Check box if met)
• EVSE gets a signed CRL from the CA/ SubCAs and validates that it is correctly signed.
• The certificate chain to issue valid EVSE and Contract certificates is operational.
• A charging session begins.

Test NameSuccessful seamless retry using EIM due to injecting V2G Comms / HLC Errors [Multiple enumerations]
Test IdentifierITP-02-06-002
Test TypeSeamless Retry
Test CategoryAdvanced Charging Tests
Test PurposeTo ensure the seamless retry is correctly implemented using EIM authentication.
Observed MetricsHLC messages, Pilot states, # retries, Session duration
Intended MRECs/Errors--
Other Possible MRECs"AuthorizationTimeout"
Test ConditionsAuthentication Type/sEIM
Plug or Authenticate firstAuthenticate
Fault Method/s1. Loss of High-Level Communication [Sr.No.14]
2. EXI encode/ decode error [Sr.No.15]
3. Error in V2G Parameter [Sr.No.16]
4. V2GTP error [Sr.No.17]
5. Wrong / Unexpected Response Msg detected by EVCC [Sr.No.18]
6. Wrong / Unexpected Request Msg detected by SECC [Sr.No.19]
7. FAILED_SequenceError Response Code received by EVCC [Sr.No.20]
8. Unprocessable response message by EVCC [Sr.No.21]
9. Unprocessable response message by SECC [Sr.No.22]
HLC Protocol/sDIN SPEC 70121, ISO 15118-2, ISO 15118-20, J3400-1
Involved System/sEV, EVSE
Test ReferenceLink to the published document for seamless retry
Steps1. Set EVSE authentication option to "Authentication Type".
2. Set TT_SR_session timer to suitable value.
3. Plug-in EV.
4. Within 30 seconds, provide "Authentication Type".
5. Inject V2G/HLC 'Fault Method' during HLC.
6. Observe EVSE for Pilot/HLC state transition after fault and retry attempt.
7. Remove 'Fault Method' within TT_SR_session timer duration to allow for a charge session to start successfully.
8. If the charge session starts, terminate it 30-60 seconds into the power transfer.
9. Unplug EV.

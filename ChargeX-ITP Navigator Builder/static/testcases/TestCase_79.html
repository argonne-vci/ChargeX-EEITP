Test NameInfinite DepartureTime duration input from EV user for CDP
Test IdentifierITP-02-10-003
Test TypeCharge Discovery Protocol (CDP)
Test CategoryAdvanced Charging Tests
Test PurposeTo check the effect of infinite DepartureTime duration provided by the EV user during the charge discovery protocol.
Observed MetricsCurrent [A(DC)], EVSE Available Current [A(DC)], EV Current Req [A(DC)]
Intended MRECs/Errors--
Other Possible MRECs--
Test ConditionsEV Initial SoC<15%
EV SoC Charge Limit100%
HLC Protocol/sISO 15118-2, ISO 15118-20
Involved System/sEV, EVSE
EV HMI charge scheduling valuesEAmount: 10 kWh, DepartureTime: 1000000000 (~Infinite duration)
Steps1. Set EVSE HLC highest priority to 'HLC Protocol'.
2. Set EVSE authentication option to 'Authentication Type'.
3. Set 'EV HMI charging schedule values'.
4. Set 'EVSE charge availability' values.
5. Plug-in vehicle. Ensure the vehicle is plugged in before any session timeout occurs.
6. Provide 'Authentication Type'.
7. Observe session initialization HLC Messaging and record "ChargingProfile" within the EV message "PowerDeliveryRequest".
8. Wait for 30 min or until the charge session terminates automatically (Whichever is shorter).
9. Unplug EV.
Pass Criteria (Check box if met)
• EV communicates Charge Parameters via a ChargeParameterDiscoveryRequest.
• EVSE responds with ChargeParameterDiscoveryResponse.
• EVSE creates the PMaxSchedule with a suitable DepartureTime and neglects the infinite value.

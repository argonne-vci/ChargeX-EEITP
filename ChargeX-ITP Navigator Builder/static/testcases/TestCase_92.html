Test NameV2G Dynamic BPT using ISO 15118-20, Command -5Amps
Test IdentifierITP-02-13-001
Test TypeDynamic Bi-directional Power Transfer (DBPT)
Test CategoryAdvanced Charging Tests
Test Purpose - To test ISO 15118-20 V2G Dynamic Bidirectional Power Transfer charging feature - To test charge scheduling capabilities within Dynamic BPT
Observed MetricsSession initialization stages, EV UI response, EVSE UI response, HLC Messaging
Intended MRECs/Errors--
Other Possible MRECsMessageSequenceFailure, Other failure
Test ConditionsHLC Protocol/sISO 15118-20
Involved System/sEV, EVSE
EV Initial SoC<50%
EV ServiceID/sServiceDiscoveryRequest:
ServiceID:17
EV Charge ParametersChargeParameterDiscoveryReq:
EVMinimumEnergyRequest: -5000W
EVMaximumDischargePower: -10000W
EVMaximumDischargeCurrent: -10A
MinimumSOC: 5%
EVSE Charge ParametersChargeParameterDiscoveryRes:
EVSEMaximumDischargePower: -10000W
EVSEMaximumDischargeCurrent: -10A
EV Bidirectional ControlDC_BidirectionalControlReq:
EVTargetCurrent: 'value'
EVMaximumDischargeCurrent: -10A
EVMaximumDischargePower: -10000W
EVSE Bidirectional ControlDC_BidirectionalControlRes:
EVSEPresentCurrent: -5A
Steps1. Set EVSE HLC highest priority to 'HLC Protocol'.
2. Set EVSE authentication option to 'Authentication Type'.
3. Set EV available service IDs to 'EV ServiceIDs'.
4. Set EV charge parameters to include 'EV Charge Parameters' values.
5. Set EVSE charge parameters to include 'EVSE Charge Parameters' values.
6. Plug-in vehicle Within 30 seconds.
7. Provide 'Authentication Type'.
8. Observe the session initialization into power transfer.
9. Ensure EV continuously sends (over HLC) 'EV Bidirectional Control'.
10. Ensure EVSE continuously responds (over HLC) with 'EVSE Bidirectional Control'.
11. If no fault occurs, after 60 seconds terminate charge session. Unplug EV.
Pass Criteria (Check box if met)
• The EV and EVSE execute a reverse Power Transfer, i.e.
• EV battery supplying the grid.
